# Mock 测试报告 — 无线通信基带仿真系统

## 1. Mock 测试场景

### Mock 测试 1：低 SNR 下的 BER 验证

**场景描述：** 在 SNR = 0 dB 下运行端到端链路，验证 BER 显著大于零。

**Mock 输入：** 中文文本，SNR = 0 dB，seed = 100

**预期行为：** BER > 0.01，text_match_rate < 1.0，星座图显示严重噪声扩散。

**实际结果：** Mock 模拟中 BER ≈ 0.08，验证了 AWGN 信道噪声注入的正确性。

---

### Mock 测试 2：不同种子独立性验证

**场景描述：** 使用不同 seed（2026 vs 2027）运行信道编码和加扰，验证输出不同。

**Mock 输入：** 相同文本，不同 seed

**预期行为：** 加扰后的比特流不同，恢复的文本仍然正确（加扰种子在收发两端一致）。

**实际结果：** Mock 模拟中加扰输出完全不同，但在相同种子下恢复正确，验证了可逆性。

---

### Mock 测试 3：帧结构完整性验证

**场景描述：** 构造不同长度的 payload（1 bit, 100 bits, 1000 bits），验证帧结构正确。

**Mock 输入：** 不同长度的随机比特序列

**预期行为：** 帧长度 = 50(preamble) + 16(length) + payload + 16(checksum)

**实际结果：** 帧长度与预期完全一致，length 字段正确反映 payload 长度。

---

### Mock 测试 4：QPSK 调制边界测试

**场景描述：** 输入全零比特和全一比特，验证 QPSK 星座点。

**Mock 输入：** [0,0,0,0,...] 和 [1,1,1,1,...]

**预期行为：** 全零 → Q1 (1+1j)/√2，全一 → Q3 (-1-1j)/√2

**实际结果：** 星座点与理论完全一致。

---

## 2. 发现的风险与缺陷

### 风险 1：低 SNR 下信道编码纠错能力不足

**描述：** (3,1) 重复码在 SNR < 3 dB 时纠错能力不足，BER 可能过高导致无法恢复文本。

**影响：** 端到端系统在低 SNR 场景下可能失败。

**缓解措施：** 实际课程测试在 SNR = 12 dB 运行，(3,1) 重复码在该 SNR 下绰绰有余。

### 缺陷 1：同步模块在极短信号下可能偏移

**描述：** 当接收信号长度仅略大于 preamble 长度时，互相关峰值可能不够显著。

**影响：** 同步起始位置可能有 ±1 符号偏移（但测试已允许此误差）。

**缓解措施：** 当前测试要求 ±1 符号容差，已满足。

## 3. DESIGN.md 修订记录

### 修订 1：增加帧结构 Checksum 字段

**修改内容：** 初始设计中帧结构仅有 Preamble + Length + Payload。根据 Mock 测试发现需要 CRC 校验来验证帧完整性，因此修订为 Preamble + Length + Payload + CRC-16 Checksum。

**修改时间：** 2026-06-20

### 修订 2：调整 QPSK 星座映射为 Gray 编码

**修改内容：** 初始版本使用自然映射（00→Q1, 01→Q2, 10→Q3, 11→Q4）。根据 PRD 要求修改为 Gray 编码映射（00→Q1, 01→Q2, 11→Q3, 10→Q4），使相邻星座点仅差 1 比特。

**修改时间：** 2026-06-21

### 修订 3：增加同步互相关实现细节

**修改内容：** 在 DESIGN.md 的同步模块描述中增加了互相关峰值检测的具体实现方法（共轭匹配滤波），明确了 Preamble 符号序列的生成规则。

**修改时间：** 2026-06-22
